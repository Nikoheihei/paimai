## 八、链路延迟对比

PlaceBid 返回前不等 WS 广播，HTTP 出价延迟与 WS 广播延迟是两条独立链路。

| 阶梯 | HTTP P50 | HTTP P99 | WS广播 P50 | WS广播 P99 |
|------|----------|----------|------------|------------|
| 50房 | 9ms | 131ms | 183ms | 720ms |
| 100房 | 30ms | 187ms | 242ms | 837ms |
| 150房 | 83ms | 570ms | 465ms | 1038ms |
| 200房 | 117ms | 1054ms | 543ms | 3204ms |

## 九、结论

- **系统错误率全程 0%**：无 5xx、无超时、无 Redis 拒绝连接
- **WS 连接成功率 100%**：2000/2000 用户全部成功建立 WebSocket
- **HTTP 出价链路本身较稳定**：P50 在 9ms~30ms~83ms~117ms，PlaceBid 返回前不等 WS 广播
- **WS 广播是独立链路**：广播延迟 P50 183ms~242ms~465ms~543ms，P99 720ms~837ms~1038ms~3204ms
- **两条链路互不阻塞**：HTTP 出价耗时不受 WS 广播影响，但 WS 广播延迟影响用户看到最新价格的实时性
- **当前主要长尾在 WebSocket 实时推送链路**，不在 MySQL、Outbox 或 Redis Stream
